# 切换租户时验证密码

> Source: https://www.yuque.com/seeyonkk/v8/rigc00e37487mpr7

作者：陈晓东

时间：2026.6.2

适用版本：5.30及以上版本

使用场景： 本场景属于标准化漏洞范畴：人员张三归属 A 租户，攻击者可在 B 租户新建同名人员账号，配置与 A 租户张三完全一致的登录账号；因系统切换租户环节无需二次核验密码，攻击者凭借该账号跨租户切换后，即可访问 A 租户下张三的业务数据

# 3、客开代码实现

说明：后端SPI配合前端插槽，后端SPI方法必须返回false,控制每个租户的密码不一致，才能达到预期的效果

## 3.1 后端SPI实现

```
/**
 *
 * 人员安全切换租户认证服务
 *
 * 配合前端切换租户功能使用，需要前端接入插槽，在人员切换身份的时候需要校验目标租户的密码是否正确
 *
 * 这个类的主要作用就是让用户可以让各个租户的密码保持不一致，只允许修改当前租户的密码，其他租户密码不能被修改
 *
 * 这样才是一个完整场景的使用
 *
 * @author liu xiong
 * @date 2026/4/19
 */
public abstract class AbstractUserSafeAuthService {
    /**
     * 是否修改所有的密码，只要认定为同一个人，就修改其在所有租户中的密码
     *
     * @return  true - 修改所有的密码； false - 只修改当前租户的密码
     */
    public abstract boolean updateAllPassword();
}
```

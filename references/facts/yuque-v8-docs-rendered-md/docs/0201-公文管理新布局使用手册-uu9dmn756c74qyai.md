# 公文管理新布局使用手册

# 1. 什么是公文新布局

公文新布局是为解决公文在复杂场景下的性能、平滑升级等一系列问题而诞生。

新布局对原有的个性化布局设计器做了重大调整，保留了公文所需的能力，去掉了部分无用且对性能产生影响的能力。

页面规则也进行了底层重构，交互逻辑由原有的控件交互模式变更为数据源交互模式，移除了部分在新架构下不支持的操作节点。

原有的标准产品页面规则均封装为标准行为动作，行为动作全局唯一，脱离原本与布局深度绑定的模式，该模式下客户bug处理与升级时实施侧无需进行任何调整【原模式下需手动修改相关页面规则】。

定制化能力方面，提供多种结合互补的定制化方式，页面规则可调用标品的行为动作进行业务前后的逻辑处理，复杂逻辑也可通过实现标品行为动作的前后插槽进行定制化业务处理，更可注册客开自有的行为动作实现更多更灵活的业务方式。

# 2. 公文旧布局启用流程

F12打开控制台，在控制台中输入“localStorage.setItem("openOldLayout", true)”，打开布局设置，选择对应页面，点击新建。以展示新设计器开关，将开关关闭后即可新建公文旧布局。

# 3. 公文新布局启用流程（已废弃-默认启用新布局）

1. 布局管理-页面URL增加如下参数：&openNewLayout=true （暂时通过URL参数屏蔽，后续正式推出后，去除URL屏蔽）。 选中对应页面，点击新建，启用新布局，修改对应名称，点击确认。

# 4. 新布局使用规范

1.  新布局基本规范【页面规则功能为5.6及后续版本支持】

a.  新布局预制控件不支持删除，如不需显示，可设置成隐藏，位置可拖动到其他位置。

b.  控件隐藏后不做渲染，原有公文隐藏控件（布局内文号/渲染组件/公文作废）已提供公文扩展事件调用控件动作。

# 7. 新布局升级规范

(1) 初次使用新布局，需要新建启用新布局，并且根据之前布局的修改和页面规则的修改，实施老师整理老布局的修改文档，根据文档的修改，在新布局下搭建实现原有布局功能。

(2) 不同租户的布局搭建，可以先实现单个租户的发文/收文/签报/签收布局的搭建，后续通过公文复用能力(详情参考大纲8:公文布局复用功能)，可把已搭建好的布局在其他租户使用，如功能一致，则可直接使用，如有部分修改，可在导入后针对性修改不同点。避免工作量重复。

(3) 当前版本新布局搭建完成后，升级后续版本(A版本-->B版本)

a. 对于直接调用公文扩展行为，使用A版本的布局，可直接使用公文后续版本(B版本)的最新公文扩展行为的功能。

b. 若自行搭建的页面规则，在页面规则内调用公文扩展行为，使用A版本的布局，可直接使用公文后续版本( B版本)的最新公文扩展行为的功能。(解决之前老布局升级版本，使用之前布局，无法使用公文新版本标品功能)

# 8. 公文业务实现机制变化

(1) 布局中原有 紧急程度 / 关联文档 / 标题 ，预置的值变化事件实现方案改变。

老布局：在布局中隐藏控件，控件绑定值变化事件。

新布局：预置字段监听机制，对应字段值变化，直接触发预置的事件。

新布局优化点：老布局之前只能是控件被用户修改时候，才可以触发，设置默认值的情况，无法触发值变化事件。新布局下可以通过设置默认值，初始化进入即可触发逻辑实现。（仅对 标品出厂的 紧急程度 / 关联文档 / 标题 字段，新布局下已删除此类控件，增加此机制用于实现原有已删除控件的值变化功能）

# 10. 公文布局显隐规则（重要‼️）

1、 优先有绑定流程操作的按钮显隐 受流程操作勾选控制，未勾选的就隐藏；

正常勾选的继续走下面的逻辑

2、 按钮受节点权限控制的，节点权限有表达式的，走节点权限表达式

3、 按钮受节点权限控制的，节点权限隐藏，则隐藏

4、 按钮受节点权限控制的，节点权限正常，则走初始化规则（后端初始化接

口initData-pageData-commponentData字段下对应控件的状态）显隐最

终结果为准

5、 按钮受节点权限控制的，节点权限为页面规则的，走页面布局的显隐配置

+ 初始化规则显隐最终结果为准， 若最终结果为正常且布局控件存在表达

式的，会走布局控件表达式显隐

6、 按钮不受节点权限控制的，走页面布局的显隐配置 + 初始化规则显隐最终

结果为准， 若最终结果为正常且布局控件存在表达式的，会走布局控件表

# 日志扩展

> Source: https://www.yuque.com/seeyonkk/v8/vu83q687l6skhpk9

作者：陈晓东

时间：2025-08-15

框架提供了灵活的日志扩展机制。定义了日志扩展接口如下：

```
public interface LogInitializer {
    void init(LogbackConfigurator configurator);
}
```

通过增加该接口的实现，可以定义扩展日志输出。例如单元测试统计日志的扩展示例如下：

1
增加LogInitializer的实现类

```java
public class UnitTestLogInitializer extends AbstractLogInitializer {
    @Override
    public void init(LogbackConfigurator configurator) {
        String fileName = Apps.getAppName() + "-test.log";
        configurator.log("设置单元测试的统计信息，日志文件为 : %s", fileName);
        Appender<ILoggingEvent> appender =
        configurator.asyncFileAppender("FILE-UNIT-TEST", configurator.getPattern(), fileName);
        configurator.logger(BootTestRunner.class.getName(), null, true, appender);
    }
}
```

在实现中，定义了单元测试统计日志的输出文件，同时指定了logger的名称跟文件输出的对应关系。

2
在spring.factories文件中增加对应的定义

```
com.seeyon.boot.log.init.LogInitializer=\
com.seeyon.boot.starter.test.log.UnitTestLogInitializer
```

3
在具体的类型利用log进行相应的输出。

```java
public class BootTestRunner extends SpringJUnit4ClassRunner {

    public void xxx() {
        // ......
        log.info("单元测试耗时， 方法[{}] 耗时：{} 毫秒", xxx);
        // ......
    }

}
```

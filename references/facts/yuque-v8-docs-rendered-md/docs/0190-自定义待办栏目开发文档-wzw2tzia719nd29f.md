## 二、实现方案分析

### 2.1 技术方案选择

由于需求高度个性化，无法在标品中直接改动，因此采用前端自定义栏目方式实现。

### 2.2 架构设计

栏目分为两部分：

●
顶部门户区域：需在标品 portal-frame 中添加搜索框

●
内容区域：由客开事项中心组件实现

搜索框的显隐由 portal-portlet/portlet-definitions 接口返回的 definitionInfo 中的新增字段 showSearchInput: true 控制。

### 2.3 标品改动：portal-frame 搜索框组件

在 portal-frame 中新增搜索框渲染逻辑：

设计要点：

●
搜索框通过 showSearchInput 配置控制显隐

●
搜索事件通过 ref 透传至事项中心

●
事项中心通过 onPortletSearch 方法执行具体搜索逻辑

### 2.4 客开实现：事项中心搜索逻辑

在客开事项中心 (ctp-affair) 中实现搜索回调：

搜索流程：

1
接收关键词并构建检索参数

2
调用全文检索接口获取匹配的事项ID列表

3
根据ID列表调用原有列表接口获取完整数据

4
更新页面状态展示搜索结果

### 3.3 配置同步（关键步骤）

#### 3.3.1 获取标品配置

新发布的栏目默认只有基础配置，需从原标品待办栏目捞取配置：

1
点击「编辑栏目」，找到 portal-portlet/portlet-definitions 接口

2
提取返回结果中的 definitionInfo → properties 字段

3
将 properties 配置复制到自定义栏目的对应位置

注意：properties 中的 groupName 和 groupId 需保留自定义栏目自身的值，仅替换 items 数组内容。

#### 3.3.2 配置跳转链接

如需修改「更多」按钮的跳转链接：

●
方式一：直接修改 moreButton 中的 path 字段

#### 3.3.3 角标配置

栏目角标通过 definitionInfo 同级的 countRule 字段控制：

1

## 四、总结

| 维度 | 说明 |
| --- | --- |
| 实现方式 | 前端自定义栏目 + 客开事项中心组件 |
| 搜索触发 | PC端通过portal-frame透传，移动端直接调用 |
| 配置来源 | 从标品栏目捞取后由后端覆盖 |
| 角标实现 | 通过 countRule 字段配置 |
| 移动端特殊处理 | 需新建页面支持磁贴添加，通过URL路径区分类型 |

#### 2.3.2 安装yarn

执行npm install -g yarn  安装yarn

#### 2.3.3 切换node源

因V8前端里面包含@seeyon/开头的组件，需要切换对应V8内网映射到外网源地址才能下载对应的组件包或对应的依赖，比如@seeyon/syf等组件；可使用命令：npm set registry http://39.107.192.235:9082/ 切换node源

#### 2.3.4 拉取udc应用前端代码

克隆udc应用代码到本地并切换到对应的设计分支版本

1）创建低代码定制平台后，如何找到对应的扩展工程代码

根据应用id进行搜索，前端自定义页面代码存放到 “autocode-extend/fronted/ ”应用id 目录

#### 2.3.5 切换到需要启动的工程目录

比如启动PC端手页页面切到PC目录，在当前目录打开cmd窗口执行yarn install --ignore-engines命令下载PC端的依赖； 启动移动端手页页面切到mobile目录，在当前目录打开cmd窗口执行yarn install --ignore-engines命令下载PC端的依赖；

#### 2.3.7 新增一个需要调试自定义页面标识的目录

因3.10版本之后扩展工程代码只存放扩展代码，udc创建的自定义页面或栏目构建成功后不会自动再对应的扩展工程里面添加对应得目录，需要在（kuozhanudczujian1880413857437925691\pc\src\pages\）目录下手动创建一个跟页面标识一样的文件夹，再将文件夹提交到跟设计版本保持一样的分支上；

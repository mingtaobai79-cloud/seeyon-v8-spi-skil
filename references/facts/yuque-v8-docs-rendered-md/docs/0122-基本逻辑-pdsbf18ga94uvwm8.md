# 基本逻辑

> Source: https://www.yuque.com/seeyonkk/v8/pdsbf18ga94uvwm8

作者：潘峰
最后更新：2026-06-09

本文主要阐述组织模型的一些规则和限定，方便理解和与三方沟通时减少沟通成本。

组织同步包含组织、岗位、职务、职级、人员。

# 1

外部部门的上级只能是外部单位或者外部部门

# 2
岗位（db表：organization.org_post）

从属于内部机构下，不是全局。比如A机构和B机构，都有开发岗，那么需要A机构和B机构下各自建一个开发岗。

岗位类型：基准岗位、自用岗位

1、基准岗位

如果三方的岗位，不区分组织，只有一套，那么可以通过基准岗位进行同步。基准岗位本质上，是平台根据所选适用范围，批量帮你复制了同样的岗位。

比如：A机构下级是B机构，那么A机构建立的岗位，数据上跟B机构是不互通的，如果A机构建立了基准岗位，且适用于与下级，那么后台除了在A机构下建立这个岗位，也会后台在B机构下建立一样的岗位。

<img width="668.5">

2、自用岗位，自用岗位就是只属于自己机构本身所用，不适用于自己的下级机构。

1
依赖于内部机构下

2
机构下的岗位code，是唯一的

# 3
职务（db表：organization.org_job）

和岗位类似，从属于内部机构下，不是全局。但是职务是组织同步中非必需项，可以不同步。

职务类型：同岗位，逻辑也一样

# 4
职级（db表：organization.org_level）

职级属于全局，不属于具体组织，跟岗位和职务不一样。

# 5
人员（db表：organization.org_member）

人员由人员基本信息表(org_member) 和 人员任职表（org_member_post）组成。

org_member表：含有人员基本信息，code、name、年龄，邮箱等

org_member_post表：包含人员任职信息，就是人员在什么单位担任什么岗位职务。

任职类型：主岗和兼职

人员停用，主岗和兼职都会停用，

人员类型：内部人员、外部单位用户、外部个人用户

1
内部人员不能放在外部组织或者线下组织下

2
外部人员不能放在内部组织下

3
主岗每人只能有一条，兼职可以多条。

4
组织+岗位+人，组成了一条任职信息，是唯一的。

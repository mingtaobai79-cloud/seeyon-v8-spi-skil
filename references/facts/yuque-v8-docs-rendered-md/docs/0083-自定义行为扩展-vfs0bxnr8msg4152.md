## 2.自定义行为扩展规范

### 2.1 package.json 规范

客开扩展行为统一使用 "@kk/action-" 开头。

### 2.2 目录结构规范

目录规范，因为syf即打包脚本采用 【契约】 的方式对 指定目录及文件 进行扫描，然后统一打包代码，需要注意的地方以 * 进行了标注。

### 2.3 扩展行为导出规范

导出文件，runtime.ts、schema.ts 规范及示例代码

runtime.ts

schema.ts

统一使用 export const  方式，以sy.config.json 中定义的 actions.key 命名函数。

导出方式 严格约定，请严格按照此方式导出。

### 2.4 schema 规范

用于导出 设计态 使用扩展行为时的参数配置表单渲染。如果该事件行为无需参数配置，则可直接导出 export default {}

#### 2.4.4 CustomComponent 自定义参数组件详解

自定义参数组件，使用 Antd Form 自定义表单控件设计思路，接受value、onChange的受控组件，渲染时会被自动包裹Form.Item。
建议使用 @seeyon/ui 基础表单组件，保持风格统一，完全自定义可能导致设计态风格迥异，影响体验。

#### 2.4.5 DynamicAttrFunction 动态属性函数详解

使用动态属性函数，可以实现属性的动态变化，目前支持动态变化的属性。
见【Schema类型定义】注解标注。注：目前仅支持必填，后续根据业务场景需要再扩大应用范围。

#### 2.4.6 Schema 导出示例

```

import { Parameter } from "./components/CustomParams";
import { onParameterChange, onParameterVisible, paramRequired } from "./utils";

export default {…};
```

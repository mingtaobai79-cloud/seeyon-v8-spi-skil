# 三方集成同步域共享资源

本目录包含三方集成同步域的共享资源和通用约束。

## 域概述

三方集成同步域包含三个 SPI 子场景：

1. **批处理三方待办** (`todo-batch/`) - 三方待办数据批量处理
2. **集成接口鉴权** (`api-auth/`) - 集成应用接口认证扩展
3. **组织同步中间表** (`org-sync-middle-table/`) - 组织数据中间表同步

## 共享约束

所有三个 SPI 都必须遵守：

1. **公共约束**：`references/spi-domain-constraints.md`
2. **域约束**：本目录下的 `constraints.md`
3. **各自 SPI 约束**：各 SPI 目录下的 `constraints.md`

## 通用规则

1. 三个 SPI 都属于"三方集成过程中的数据/请求同步和处理"
2. 生成代码时必须分三个独立模块/包
3. 所有 SPI 都涉及与三方系统的交互，必须做好异常隔离和日志记录
4. 敏感信息（token、secret 等）禁止明文打印

## 依赖版本

- `ctp-affair-facade:5.3.315`（批处理三方待办，已通过用户现场 artifact 核验；3.15.0 为文档最低适用线）
- `cip-connector-api:3.8.211`（集成接口鉴权、组织同步中间表；组织同步版本待现场最终标定）

## 重启服务

- `ctp-affair`（批处理三方待办）
- `cip-connector`（集成接口鉴权、组织同步中间表）
